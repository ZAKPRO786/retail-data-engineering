# Retail Data Engineering Pipeline for Business Insights Generation

## Overview

This repository contains an end-to-end Retail Data Engineering solution developed for ABC Retail Solutions. The project demonstrates the complete lifecycle of retail data processing, including ingestion, data quality validation, transformation, standardization, governance, KPI generation, and business intelligence reporting.

The solution converts raw transactional data from multiple retail channels into curated, analytics-ready datasets and interactive Power BI dashboards that support data-driven decision-making.

---

## Business Problem

ABC Retail Solutions receives retail transaction data from multiple operational systems. Due to differences in data collection and source formats, the raw datasets contain several data quality issues that affect reporting accuracy and business analysis.

### Key Challenges

* Missing and incomplete records
* Duplicate transactions
* Inconsistent product names
* Inconsistent category classifications
* Multiple date formats
* Invalid quantity values
* Exposure of Personally Identifiable Information (PII)
* Lack of standardized reporting datasets

The objective of this project is to design and implement a robust data engineering pipeline capable of producing reliable, business-ready data assets.

## Data Engineering Workflow

### 1. Data Ingestion

* Load retail_data1.csv
* Load retail_data2.csv
* Load product_details.csv
* Validate schema consistency
* Merge datasets

### 2. Data Cleaning

* Remove duplicate records
* Handle missing values
* Validate quantity values
* Enrich records using product master data

### 3. Data Transformation

* Merge business datasets
* Calculate revenue metrics
* Generate reporting attributes
* Prepare analytical datasets

### 4. Data Standardization

* Standardize product names
* Standardize category values
* Normalize date formats

### 5. Data Governance

* Mask customer email addresses
* Mask customer phone numbers
* Generate governance-ready outputs

### 6. KPI Generation

The pipeline calculates:

* Total Revenue
* Total Orders
* Total Customers
* Average Order Value
* Revenue by Category
* Revenue by City
* Revenue by Product
* Revenue by Payment Method

Revenue Formula:

```text
Revenue = Price × Quantity × (1 - Discount)
```

Relationship:


## Quick Start

Clone the repository:

```bash
git clone <repository-url>
cd Retail_Data_Engineering
```

Create virtual environment:

```bash
python -m venv venv
```

Activate environment:

```bash
venv\Scripts\activate
```

Install dependencies:

```bash
pip install -r Code/requirements.txt
```

Run pipeline:

```bash
python Code/retail_pipeline.py
```

## Assumptions

* Product master data is considered the trusted source.
* Missing product attributes can be derived using product_id.
* Invalid quantities represent data-entry errors.
* Masked customer information is sufficient for reporting and analytics.
